# <Brand /> CLI

The CLI is the direct command-line client and the runtime service host used by editor clients.

## Install

```sh
npm install -g @truss-harness/cli
```

The global installer prints the first commands to run. `truss-cli` and `truss-cli help` display the complete built-in reference without requiring a model.

## Commands

| Command | Purpose |
| --- | --- |
| <CliCommand /> `chat <prompt>` | Stream a one-shot agent task |
| <CliCommand /> `models` | List detected local endpoints and models |
| <CliCommand /> `config path` | Print user and workspace configuration paths |
| <CliCommand /> `config init` | Create a workspace profile template |
| <CliCommand /> `init` | Generate or refresh repository instructions |
| <CliCommand /> `update [note]` | Store Git state and a handoff note |
| <CliCommand /> `status` | Inspect durable workspace state |
| <CliCommand /> `clear-memory` | Reset durable workspace memory |
| <CliCommand /> `commands` | Print shared slash commands |
| <CliCommand /> `serve` | Host the JSONL runtime protocol |

## Workspace commands

Run <CliCommand /> `init` to create or refresh generated repository context in `AGENTS.md`. Use <CliCommand /> `update [note]` to record Git state and a handoff note in durable memory, <CliCommand /> `status` to inspect it, and <CliCommand /> `clear-memory` to reset it. These commands do not need a configured model. <CliCommand /> `commands` prints the same slash-command help used by the TUI and VS Code panel.

## Chat

`chat` creates an in-memory session, streams output to standard output, and prints tool requests to standard error.

<CliUsage command={'chat --profile ollama-coder --mode plan "Find the files that define the agent loop"'} />

Direct CLI chat auto-allows tools. Use it only in trusted workspaces. The terminal UI and VS Code client provide interactive or policy-based approvals.

Add `--internet-access` to expose `web_search` and `web_fetch` for that run. Because direct CLI chat auto-allows tools, enabling this flag also authorizes public network requests. Use `--no-internet-access` to override an enabled profile.

When a Plan-mode run produces a checklist, the CLI prints the saved plan to standard error. The active plan is stored at `.truss-harness/plans/active.json` and is automatically included in later Edit-mode context.

## Service mode

<CliUsage command="serve --profile ollama-coder" />

Service mode uses newline-delimited JSON on standard input/output. Output is reserved for protocol messages; do not add logs to standard output. The VS Code extension starts this service automatically.

## Model discovery

<CliUsage command="models" />

This prints detected endpoints and model IDs for local servers.
